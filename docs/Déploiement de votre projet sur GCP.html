Déploiement de votre projet sur Google Cloud Platform

Guide complet :

Ce guide vous accompagne pas à pas pour installer, configurer et déployer un projet Python avec Docker et MongoDB sur une machine virtuelle (VM) Google Cloud Platform (GCP).

Étape 1 : Créer et accéder à une VM sur GCP

  1. Connectez-vous à votre compte GCP et allez dans Compute Engine > VM Instances.
  1. Cliquez sur Créer une instance et configurez :
    • Nom : vm-mongo-migration.
    • Type de machine : e2-medium (2 vCPU, 4 Go de RAM).
    • Système d'exploitation : Ubuntu 20.04.
    • Autorisez le trafic HTTP et HTTPS.
  1. Cliquez sur Démarrer.
  1. Une fois la VM créée, accédez-y en cliquant sur le bouton SSH.

Étape 2 : Préparer la VM

Installer les outils essentiels

  1. Mettre à jour la machine :
    sudo apt update && sudo apt upgrade -y
  1. Installer Git :
    sudo apt install -y git
    • Vérifiez l'installation :
      git --version
  1. Installer Python et pip :
    sudo apt install -y python3 python3-pip
  1. Configurer les règles de pare-feu pour MongoDB :
    • Accédez à VPC Network > Règles de pare-feu dans le menu GCP.
    • Cliquez sur Créer une règle de pare-feu et configurez :
      • Nom : allow-mongodb.
      • Réseau : Par défaut (ou celui de votre VM).
      • Cibles : Toutes les instances ou spécifiez votre VM.
      • Filtres IP source : Limitez l'accès à une plage IP (par exemple, uniquement votre IP publique).
      • Protocoles et ports : TCP, port 27017.
    • Enregistrez la règle et vérifiez qu'elle est appliquée.
  1. Installer Docker :
    sudo apt install -y docker.io
sudo systemctl enable --now docker
sudo usermod -aG docker $USER
    • Redémarrez votre session SSH pour appliquer les changements.
  1. Installer Docker Compose :
    sudo curl -L "https://github.com/docker/compose/releases/download/v2.22.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
    • Vérifiez l'installation :
      docker-compose --version

    • Si votre dépôt est privé, configurez une clé SSH pour éviter de saisir vos identifiants à chaque clonage.

Étape 4 : Configurer un environnement virtuel Python

  1. Vérifier la version de Python installée :
    • Assurez-vous que Python 3.6 ou une version ultérieure est disponible sur votre système :
      python3 --version
    • Si Python n’est pas installé, suivez les étapes dans la section précédente pour l’installer.
  1. Installer le package python3-venv :
    • Ce package est nécessaire pour créer des environnements virtuels. Installez-le avec :
      sudo apt install -y python3-venv
    • Si vous rencontrez une erreur, assurez-vous que le système est bien mis à jour et que Python est correctement configuré.
  1. Créer un environnement virtuel :
    • Naviguez dans le répertoire de votre projet :
      cd ~/migration-donnees-mongodb
    • Créez un environnement virtuel :
      python3 -m venv venv
    • Si une erreur indique que le module venv est manquant, assurez-vous que python3-venv est bien installé.
  1. Activer l'environnement virtuel :
    • Activez l'environnement avec :
      source venv/bin/activate
    • Une fois activé, vous devriez voir (venv) apparaître au début de votre ligne de commande.
  1. Installer les dépendances :
    • Installez les bibliothèques nécessaires listées dans le fichier requirements.txt :
      pip install -r requirements.txt
    • Si une erreur survient, vérifiez que votre environnement est bien activé et que pip est à jour :
      pip install --upgrade pip
  1. Installer le package python3-venv
    • Vérifiez d'abord la version de Python disponible sur la VM pour vous assurer qu'elle est compatible avec venv :
      python3 --version
      • Python 3.6 ou une version ultérieure est requise.
    • Installez le package python3-venv avec la commande suivante :
    sudo apt install -y python3-venv
  1. Créer un environnement virtuel :
    python3 -m venv venv
  1. Activer l'environnement virtuel :
    source venv/bin/activate
    • Vous devriez voir (venv) au début de votre ligne de commande.
  1. Installer les dépendances :
    pip install -r requirements.txt

Étape 5 : Lancer MongoDB avec Docker Compose

  1. Vérifier la présence du fichier docker-compose.yml :
    ls
    • Assurez-vous que docker-compose.yml est dans le répertoire courant.
  1. Lancer Docker Compose :
    docker-compose up -d
    • Pour vérifier les conteneurs en cours d'exécution :
      docker ps
  1. Dépanner les problèmes de permissions Docker :
    • Si vous rencontrez une erreur comme "permission denied", ajoutez votre utilisateur au groupe Docker :
      sudo usermod -aG docker $USER
      • Déconnectez-vous et reconnectez-vous pour appliquer les changements.
    • Vérifiez que le groupe Docker a les bonnes permissions sur le socket Docker :
      ls -l /var/run/docker.sock
      • Si nécessaire, corrigez-les :
        sudo chown root:docker /var/run/docker.sock
sudo chmod 660 /var/run/docker.sock
  1. Vérifier la présence du fichier docker-compose.yml :
    ls
    • Assurez-vous que docker-compose.yml est dans le répertoire courant.
  1. Lancer Docker Compose :
    docker-compose up -d
    • Pour vérifier les conteneurs en cours d'exécution :
      docker ps

Étape 6 : Exécuter le projet

  1. Vérifier que les fichiers requis sont présents dans le conteneur :
    docker exec -it <nom_du_conteneur> ls /app/scripts
docker exec -it <nom_du_conteneur> ls /app/data/processed
  1. Exécuter le script Python dans le conteneur :
    docker exec -it python_application_container python /app/scripts/main.py /app/data/processed/healthcare_dataset_cleaned.csv

Étape 7 : Sauvegarder et éteindre la VM

  1. Sauvegarder les modifications sur GitHub :
    git add .
git commit -m "Modifications"
git push
  1. Arrêter la VM pour économiser des coûts :
    • Depuis la console GCP, allez dans Compute Engine > VM Instances.
    • Cliquez sur Arrêter à côté de votre VM.

1. Vérifier si un environnement virtuel est créé

Un environnement virtuel est généralement un dossier nommé venv (ou un autre nom personnalisé) dans votre projet.

  1. Lister les fichiers dans votre répertoire actuel :
    ls
    • Recherchez un dossier nommé venv (ou similaire).
    • Si vous ne le voyez pas, vérifiez les dossiers cachés avec :
      ls -a
  1. Vérifiez le contenu de l’environnement virtuel :
    • Si le dossier venv existe, regardez son contenu pour confirmer :
      ls venv
    • Vous devriez voir des sous-dossiers comme bin, lib, et include dans un environnement virtuel valide.

2. Vérifier si un environnement virtuel est activé

Lorsque l’environnement virtuel est activé :

  • Vous verrez son nom affiché au début de chaque ligne de commande entre parenthèses. Par exemple :
    (venv) user@machine:~/project$

Si ce n’est pas visible :

  1. Vérifiez manuellement l’activation avec la commande suivante :
    which python
    • Si l’environnement virtuel est activé, le chemin affiché pointera vers un Python dans le dossier venv. Par exemple :
      /path/to/project/venv/bin/python
  1. Si le chemin pointe vers /usr/bin/python ou similaire :
    • Cela signifie que l’environnement virtuel n’est pas activé.

3. Activer un environnement virtuel

Si l’environnement virtuel existe mais n’est pas activé :

  1. Activez-le avec la commande suivante :
    source venv/bin/activate
    • Pour Windows :
      venv\Scripts\activate
  1. Vérifiez qu’il est activé :
    • Le prompt de commande affichera (venv) au début de la ligne.

4. Si aucun environnement virtuel n’existe

Si aucun environnement virtuel n’a été créé :

  1. Créez-en un :
    python3 -m venv venv
  1. Activez-le comme indiqué ci-dessus.

Résumé

  • Pour savoir si un environnement est créé : Cherchez un dossier venv dans votre projet.
  • Pour savoir s’il est activé : Regardez si (venv) apparaît dans votre ligne de commande ou utilisez which python.
